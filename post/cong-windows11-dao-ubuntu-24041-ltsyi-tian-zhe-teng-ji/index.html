从Windows11到Ubuntu 24.04.1 LTS，一天折腾记

2025-01-12

1月10号
这天我正在反反复复折腾一台老东西——红米3S。有关这台机器的故事暂不多说，总之我是被它玩了，之后突发奇想，把放镜像的U盘格式化然后打开ubuntu网站下载最新的桌面版。下载文件后开始烧录，烧录后却出现两个盘符，此时打不开u盘了，且不能格式化，会提示“该u盘写有保护”。我百思不得其解，“按照视频一步步走，怎么会出错”。
1月11号
网上查找教程，下载rufus烧录一遍，成功。
我的bios是传统启动方式，磁盘是mbr分区，我先为ubuntu预留40G未分配空间，然后插上u盘进入bios更改启动选项，u盘启动放到第一个，f4保存并退出。
此时会出现一个选单，选择第一项试用&安装ubuntu，
进入ubuntu后耳目一新，接下来点选安装ubuntu，开始安装配置，到最后一项该选择第一个或最后一个(安装双系统)。
选第一个会与windows共存，系统自动选择未分配空间分区。
选最后一个，自己分区。
我选第一个，完成时区选项后开始安装，安装失败可能是空间不足。
安装成功，点击重启，出现英文提示，拔出u盘并按回车键。
重启后没出现选择系统选项，而是直接重启到ubuntu。我通过修改grub上默认启动项为4，重新启动，就回来windows了。grub选项还是不显示，这时我又重装ubuntu，但这是错误行动，正确的方法如下：
打开终端，输入sudo add-apt-repository ppa:yannubuntu/boot-repair && sudo apt-get update回车，sudo apt-get install -y boot-repair && boot-repair
选择第一项，这个软件就开始自动修复引导，重启后终于能看到grub选项了，选择第四项，可以正常回到windows。
1月12号
安装软件要输入命令，今天已经安装了vscode、gridea、clashvergerev、chrome、steam，软件体验不错，该有的基本都有。
